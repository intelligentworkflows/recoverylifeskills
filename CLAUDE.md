# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**recoverylifeskills.com** is a static informational website about 12-step recovery from alcoholism and substance abuse, written from the author's personal first-hand perspective (9+ years sobriety). The site targets people who cannot commit to traditional 90-day rehab programs and need practical, low-commitment entry points into recovery.

## Build System

Pages are generated by `build.py` from templates. **Never edit generated `index.html` files directly** — changes will be overwritten on the next build.

```
python build.py          # generate all pages
python build.py --check  # verify all generated files are in sync (exits 1 if not)
```

### How it works

- **`_templates/`** — source templates, one per page. The root page is `_templates/index.html`; all sub-pages use `_templates/<dir>/page.html`.
- **`_partials/`** — shared fragments injected at build time:
  - `_partials/disclaimer.html` — the legal disclaimer block rendered at the top of every page
  - `_partials/footer.html` — the site-wide footer with all nav links
- Templates use two placeholders: `{{disclaimer}}` and `{{footer}}`. The build replaces them, resolving `{{base}}` inside partials to the correct relative path (`../` for sub-pages, empty for root).
- Output goes to `index.html` at the root or `<dir>/index.html` for sub-pages.
- The build prefixes every generated file with a `<!-- GENERATED FILE -->` comment pointing back to its template source.

### Adding a new page

1. Create `_templates/<new-dir>/page.html` using `{{disclaimer}}` and `{{footer}}` — never hard-code the disclaimer or footer HTML.
2. Add the nav link to `_partials/footer.html` (propagates to all pages automatically).
3. Add the nav link to the `<nav class="toc">` in `_templates/index.html`.
4. Run `python build.py`.

To preview locally, open any generated `index.html` in a browser, or serve with:
```
python -m http.server 8080
```

Deployment is manual: copy all generated `index.html` files and `style.css` to the web host.

## Architecture

### File layout

```
_partials/
  disclaimer.html     shared disclaimer block ({{base}} placeholder for relative paths)
  footer.html         shared footer with all page links ({{base}} placeholder)
_templates/
  index.html          root page template
  <dir>/page.html     sub-page templates
_draft/
  <dir>/page.html     unpublished pages — not built
<dir>/
  index.html          generated output — do not edit
index.html            generated output — do not edit
style.css             single shared stylesheet — edit directly
build.py              generator script
```

### CSS Design System

All theming uses CSS custom properties defined in `:root` in `style.css`:
- Colors: `--bg`, `--surface`, `--border`, `--text`, `--muted`, `--accent`, `--accent-light`, `--border-blue`
- Layout: `--max-w` (680px content width), `--radius` (6px)
- Font: system stack (SF Pro Display → Segoe UI → sans-serif)

Reusable component classes:
- Callouts: `.note-box`, `.warning-box`, `.urgent-box`, `.checkin-box`, `.disclaimer`
- Lists: `.step-list`, `.sub-list`, `.activity-list`, `.recording-links`
- Cards: `.card-grid` + `.success-card` (two-column grid), `.organ-cards` + `.organ-card` (stacked, organ-damage page only)
- Timeline: `.timeline` and its `.timeline-entry` / `.timeline-period` / `.timeline-title` / `.timeline-detail` children — used by the home page only
- Quotes: `.bb-passage` (Big Book), `.verse-quote` / `.verse-cite` (scripture)

Collapsible sections use native `<details>`/`<summary>` with CSS-only +/- toggle animation.

| `successes/` | Things That Worked — strategies behind long-term sobriety |

`_draft/` holds pages pulled from the live site (the per-week timeline pages, `upon-awakening/`, `triggers/`, `avoid/`, `faq/`, and others). The build ignores it — it only globs `_templates/`. Move a directory back into `_templates/` to republish, and re-add its footer and home-page nav links.

### External Dependencies

Apple Podcasts iframes embed Joe & Charlie's "Big Book Comes Alive" series across multiple pages. No other third-party resources — no analytics, no tracking, no cookies (by design).

## UI Business Rules

These are non-negotiable constraints for all pages and any future changes:

- **No cookies** — no tracking, no session storage, no localStorage, no analytics scripts.
- **No images** — convey all information through text and typography alone.
- **Mobile-first** — design for small screens first; desktop legibility is required but secondary.
- **FAQ pattern** — content is presented as questions that expand to reveal answers. Use native `<details>`/`<summary>` for all expandable Q&A. Avoid decorative chrome or sidebars that distract from the content.
- **Readable typography** — large base font size, generous line height, short line length (`--max-w: 680px`). Audience is non-technical and may be in crisis; clarity always beats cleverness.
- **Minimal visual noise** — no hero sections, no cards for decoration, no icons, no animations beyond the expand/collapse toggle.

## Content Guidelines

- The site is anonymous — no author identity, no contact form, no user accounts.
- All content reflects the AA Big Book and 12-step tradition; maintain that voice when editing.
- The disclaimer block at the top of every page is legally significant — do not remove or alter it without explicit instruction. In templates it appears only as `{{disclaimer}}`.
- Quoted AA literature and Bible passages must never be paraphrased or altered — their wording is legally safe and doctrinally significant.
